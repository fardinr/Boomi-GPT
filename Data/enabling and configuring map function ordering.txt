map function ordering lets you control the order in which map functions are applied. map function ordering is useful, for example, when you need to store a value in a process property and use a different function to extract that value. map function ordering may also be useful with specialized summing functions using process properties or external systems. procedure in a map to which you have added functions, click order function executions in the functions column. the map function ordering dialog opens. select enable map function ordering?. a list of the functions appears. in the list, drag and drop reorder icons for functions until the list is in the desired order. as you drag a reorder icon, a dashed horizontal line dynamically shows the position into which the function would be moved if you were to drop the icon at the pointer location. click ok.
