override the default of the base url on the shared web server panel within atom management. before you begin you must have the atom management privilege to perform this task. if you have the atom management read access privilege, you cannot perform the task. about this taskif you want to use the shared web server on an atom that is behind a nat (network address translation) or some other network forwarding device, you can choose to manually override the url that is used when building the wsdl. to get the correct location into the wsdl, do the following. procedure on the manage menu select atom management. select the atom from the list on the left. under settings & configuration, click shared web server. on the general tab do the following: for base url for api requests, select override default. type the url in the field provided. for example: https://<external/internal host>[:<port>] in the listening port configuration section, ensure that the port that you want to configure is enabled. if the external url is https and the internal url is http, do the following in the advanced settings section: specify an external host in the external host field. in the listening port configuration section, set the port’s external port field to a number greater than 0. the typical external ssl port is 443. in the listening port configuration section, turn on the port’s external ssl check box. click save to apply the settings on all of the shared web server panel tabs. you are prompted to restart the atom, molecule, or atom cloud. you have the option to restart now or restart later.
