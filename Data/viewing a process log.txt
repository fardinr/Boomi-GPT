you can view a process log in process reporting. procedure select manage > process reporting. find the desired process execution result. do one of the following: click the view process logs icon for that execution. click the date/time link for that execution to go to its detail view and in the actions menu, select view process logs. note: if a process does not execute, because it was discarded or because the user tried to execute it when it was already running, a log is not created. if that is the case, the view process logs icon is not available. the show log dialog opens and the boomi platform connects to the atom and retrieves the log's contents. once retrieved, the logged events are shown in the dialog. optional: to download the contents of the log, click save logs. depending on your browser and its download settings, you may be prompted to specify the destination path and filename. the default filename is process_log.zip. to return to the view from which you opened the dialog, click cancel.
