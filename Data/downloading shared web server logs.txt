you can download shared server log files if your account owns the local atom, molecule, or atom cloud, or if your account has access to the private atom cloud. about this taskthe server logs that you download are specific to your account. procedure go to manage > atom management. select the atom, molecule, or cloud from the list on the left. in the log files panel, select the date of the logs that you need from the calendar or use the format yyyy-mm-dd. a valid date is no later than the current date and no earlier than the latest purge date. if you select a date outside the valid date range, a warning message appears. you can change the date or keep it. if no logs are available for the selected date, a message will appear after you click download. the dates and times in these log files are based on the jvm’s default time zone. select shared server logs. click download. the log files are downloaded to the default directory that your browser uses. the dates and times in these log files are based on the jvm’s default time zone. open or save the file.
