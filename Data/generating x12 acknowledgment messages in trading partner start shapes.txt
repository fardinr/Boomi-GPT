follow these steps to configure an x12 trading partner to enable generation of acknowledgment messages in trading partner start shapes: procedure in the trading partner component, select the x12 standard tab. expand basic x12 options. do one of the following: to enable generation of functional acknowledgment (fa) messages at the functional group level, in the functional acknowledgment options list, select acknowledge functional groups. to enable generation of fa messages at the transaction set level, in the functional acknowledgment options list, select acknowledge transaction sets. note: you may also opt to set document envelope option, element delimiter, and segment termination character. select the document types tab. for each transaction set for which you want to generate message-level acknowledgments, do the following: click add document type. in the select transmission dialog, select the transaction set and click ok. the document options panel appears. optional: to enable generation of 999 fa messages instead of 997 fa messages, select use 999 acknowledgment. optional: to enable generation of ta1 interchange acknowledgment messages, select use ta1 acknowledgment. optional: if you are enabling x12 document validation and validation rules are defined for segments in the document profile against which validation is to be performed, to enable the rules to be applied to the segments in documents, select the profileâ€™s options tab and select segment validation rules. select the document profile to use for document validation. click close. click save or save and close.
